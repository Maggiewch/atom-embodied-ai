# Chef Robotics · 档案

> 用**食品工厂的备餐装配**起家——AI 机械臂（ChefOS）在高混合产线上把**会变形、又黏又湿**的食材分装进餐盒。
> 它靠「部署即采数据」攒下号称全球最大的真实食品操作数据集，**至今独立、还在加速**。
> **这是「数据飞轮楔子打法」的正面样本，也是 [Covariant](covariant.md) 的镜像对照。**

## 1. 创始团队与基因
**Rajat Bhageria——连续创业者，自上而下选赛道。** Wharton 经济学 + Penn 机器人/ML 硕士；前作 ThirdEye（为视障者描述环境的计算机视觉产品，~2017 售出）、Prototype Capital（投「老行业 + 新技术」的 pre-seed 基金）。

基因 = **应用 AI + 市场反推**。他直接从美国劳工统计局数据反推：食品是美国**第三大劳动力**、且是**当前 AI 操作最可处理（tractable）的最大市场**（护理、零售劳动力更大但更难做）。这种「先从账本和缺工反推、再选最容易啃的硬骨头」的打法，是它楔子之所以锋利的根。

## 2. 技术路线（模仿学习 + 扩散策略，刻意区别于刚体 VLA）
- **ChefOS**：通用食品操作模型，自称「物理世界的 ChatGPT」，跑 **See → Think → Act** 闭环（深度/RGB 视觉识别食材与餐盒 → 学习策略 → 操作）。
- **路线 = 模仿学习（learning-from-demonstration）+ Transformer 扩散策略**，类比 Sora 但输出的是**机器人动作**而非像素。**不是 sim-RL，也不是语言可控的通用 VLA。** 新食材靠人**演示**上手，不改代码。
- **Food Foundation Model（2026-05）**：单一模型替代多个任务专用模型，**明确对标并区别于刚体 VLA**——「市面 VLA 多在刚体操作上训练，而食品是高变异、可变形（湿/黏/不规则）的材料」。支持**跨本体迁移**（不同运动学/末端执行器），路线图含**零/少样本上新食材**与**双臂**灵巧装配。
- **为什么食品先做**：可变形食材**仿真做不好、互联网也爬不到**，所以唯一可行的训练场就是真实产线——这把「数据瓶颈」直接变成了护城河。

## 4. 商业模式与客户
- **RaaS = 固定年费订阅**（**不是按产量付费**），打包硬件 + 软件 + 维护 + 7×24 监控 + 升级，客户**零 CapEx**。Chef 自己用**设备融资债**（SVB）出资买机器人车队——这正是重资产创业公司不靠稀释股权也能扩部署的金融机制。
- **半自主，非无人**：产线生产时机器人自主跑（视觉 + 力觉 + 称重托盘控分量），但换班/换产时需人**选餐品、装指定夹具**，且人与机器人同线协作。
- **ROI 三杠杆**：人力（缺工 + ~150% 流动率 + 冷库工伤）、产量、减少超量给料（giveaway）。话术：「Chef 的成本不变，而人力成本年年涨」。
- **客户（已核实）**：Amy's Kitchen（首客 2022）、Sunbasket、Chef Bombay、Cafe Spice、CookUnity、Revolution Foods、F&S Fresh Foods、Avatar Foods、Project Open Hand；另有未具名的「全美最大校餐供应商之一」「全球最大航食公司之一」。
  > ⚠️ 核实纠偏：网传的 **Saffron Road、Cuisine Solutions 查无实据**，不计入。

## 7. 拨开迷雾（争议与真实进度）
- **份数 ≠ 营收**：「1 亿份」是数据/部署规模指标，公司不披露营收、ARR、估值、RaaS 单价或回本期，只说一年内见 ROI。
- **「最大数据集」是自述**：「全球最大真实食品操作数据集」「比所有同行加起来还多一个数量级」是公司主张，无第三方审计。
- **泛化有边界**：跨 ~2000 种食材与同种食材的批次差异确实强，但据 IEEE Spectrum，**全新的操作类型仍需新的真实数据**，不是无限零样本。
- **自主度仍是半自主**：换产/换班要人工，2024 年的口径，可能随 Chef+ / 机间协同逐步改善。
- **路线诚实**：它没硬蹭「通用 VLA」，而是承认食品要专门的可变形操作模型——这种不吹大词、对着真实约束做的态度，反而是加分项。

## 8. 用打分卡量一量（[→ 07 判断打分卡](../智能机器时代/07-判断打分卡.md)）
| 轴 | 评分 | 依据 |
|---|---|---|
| **泛化** | ★★★★☆ | 跨 ~2000 种食材 + 批次差异 + **可变形食品（最难的操作）**；但新操作类型仍需新数据，非无限零样本 |
| **几个9** | ★★★★☆ | 选了**容错楔子**：分量错了人能补、半自主即可上线 → **不需要很高的9就能出货**，这正是它能跑通的原因 |
| **数据效率** | ★★★★★ | **演示上新、不改代码**；部署即采到别人拿不到的可变形食品数据 → 复利护城河最硬 |

> **楔子判定：真在推进（教科书级）。** 它满足楔子全部要件——人招不到（缺工 + 150% 流动）、传统自动化做不了（高混合 + 可变形）、失败便宜可恢复（半自主即可）、且**楔子里产出别人拿不到的专有数据**。
> **与 [Covariant](covariant.md) 的镜像对照**：两家跑的是**同一套楔子 + 飞轮打法**。差别在结局——Covariant 的楔子是亚马逊的命脉，被逆向收购掏空；**Chef 的楔子（食品制造）背后没有科技巨头在位者**，于是它能保持独立、把飞轮越滚越大。**给 PM 的一课：好的楔子不仅要三轴过关，还要选在「没有能把你买走的巨头」的领域里。**
