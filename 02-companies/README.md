## 一、玩家分型（Archetype）

具身智能公司按「在价值链的卡位」分四类，这是看懂一家公司的第一个坐标：

| 类型 | 定义 | 代表 |
|------|------|------|
| **🦾 本体 OEM** | 做硬件本体，大脑外采或半开放 | 宇树、傅利叶、Boston Dynamics、加速进化 |
| **🧠 大脑 / VLA** | 只做具身大模型与数据，不做（或弱化）本体 | Physical Intelligence、Skild AI、Gemini Robotics |
| **🔧 上游零部件** | 减速器、丝杠、力传感器、灵巧手、芯片 | 绿的谐波、因时机器人、坤维、地平线 |
| **🏗️ 全栈一体** | 本体 + 大脑自研闭环 | Tesla、Figure、智元、银河通用、1X |

> 卡位决定商业模式与生死逻辑：本体厂拼成本与供应链；大脑公司拼数据飞轮；
> 上游拼良率与产能；全栈拼资本厚度与节奏。

## 三、档案标准模板

每份档案统一用下列结构（见 `_TEMPLATE.md`）：

1. **速览卡** — 成立时间 / 总部 / 创始人 / 类型 / 最新融资与估值 / 核心产品 / 员工数
2. **一句话定位** — 这家公司在价值链卡哪个位，靠什么吃饭
3. **创始团队与基因** — 创始人背景决定技术路线与文化
4. **技术路线** — 本体规格 / 大脑（VLA）/ 数据策略 / 自研 vs 外采
5. **产品与里程碑** — 时间线，关键发布
6. **商业模式与客户** — 怎么赚钱、真实客户、订单、是卖货还是卖故事
7. **落地案例** — 标注「自主 / 半自主 / 遥操作 / 演示」
8. **融资与资本** — 历轮融资、投资人、估值、现金消耗判断
9. **拨开迷雾** — Demo 与现实的差距、真实进度、风险、争议
10. **信源** — 关键数据出处与日期

> 命名：`02-companies/<公司英文小写-连字符>.md`，如 `figure-ai.md`、`unitree.md`。
