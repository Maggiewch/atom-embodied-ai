## 一、我们怎么看这个行业（方法论 / 第一性原则）

1. **区分「Demo」与「部署」。** 大量惊艳视频是**遥操作（teleoperation）**或剪辑产物。
   每个案例必须标注：自主（autonomous）/ 半自主 / 遥操作 / 演示。这是拨开迷雾的第一刀。
2. **跟着钱走。** 区分三种「收入」：真实商业收入、试点/补贴收入、以及 VC 资本本身。
   一家公司是在卖产品，还是在卖故事？单位经济学（unit economics）能不能闭环？
3. **硬件是命门，数据是飞轮。** 软件（VLA 大模型）最性感，但成本能否量产取决于上游硬件
   （丝杠、谐波减速器、力传感器、灵巧手）；能力能否泛化取决于数据飞轮能否转起来。
4. **一手优先，标注信源与时间。** 每条关键数据标注来源与日期；估值/融资等易变数据定期核查。
5. **承认不知道。** 不确定的地方明确标「待核实」，而不是用确定的语气糊过去。

## 二、行业全景地图（价值链：上游 → 中游 → 下游）

```
┌─────────────────────────── 上游 · 零部件与算力 ───────────────────────────┐
│  执行/传动：谐波减速器 · RV减速器 · 行星滚柱丝杠 · 无框力矩电机 · 伺服      │
│  感知：六维力/力矩传感器 · 编码器 · 触觉传感器 · 视觉/激光                  │
│  灵巧手：欠驱动 / 全驱动手指、腱传动                                        │
│  算力：训练GPU · 端侧推理芯片（Jetson Thor / 地平线 / 寒武纪）             │
│  能源：电池 · 电源管理                                                      │
└────────────────────────────────────────────────────────────────────────┘
                                   ↓
┌─────────────────────────── 中游 · 本体 + 大脑 ────────────────────────────┐
│  本体（Hardware OEM）：人形 / 四足 / 双臂 / 移动操作平台                    │
│  大脑（VLA / 具身大模型）：端到端控制、世界模型、数据采集与仿真             │
│  全栈玩家：本体 + 大脑一体（Tesla / Figure / 智元 / 银河通用 …）           │
└────────────────────────────────────────────────────────────────────────┘
                                   ↓
┌─────────────────────────── 下游 · 应用场景 ──────────────────────────────┐
│  工业制造（汽车厂、3C）· 物流仓储 · 商用服务 · 科研教育 · 家庭消费（圣杯） │
└────────────────────────────────────────────────────────────────────────┘
```

**当前真实状态（2026 上半年判断，待持续修订）**
- 「ChatGPT 时刻」尚未到来；核心瓶颈是**数据**，不是本体。
- 真金白银的收入主要来自：科研/教育整机、少量工业试点、以及资本本身。
- 卡脖子在上游：行星滚柱丝杠、六维力传感器、灵巧手、谐波减速器。
- 家庭场景是圣杯，也是最难；工业/物流是最现实的第一落点。

## 五、维护机制（保持鲜活）

这是一座**活**的知识库。易变数据（估值/融资/订单/落地）会过期，约定如下维护纪律：

- **易变字段标注**：估值、融资、在役台数等一律标注**日期或「待核」**，绝不把旧数当现状。
- **定期 freshness 核查**：对旗舰易变事实（头部融资轮、IPO 进度、重大订单/落地）定期联网复核，更新档案 + 本日志。
- **优先级**：先核「会改变判断」的（如 IPO 落地、巨额新轮、盈利转向），再补增量细节。
- **新玩家入库**：先进 `01-landscape` 全表与 `02-companies` 观察名单，按优先级再建档。
- **每次更新都留痕**：改了什么、依据什么、日期——都记进对应档案信源与本日志。

**自动追踪机制**（`_maintenance/`）：
- [watchlist.md](_maintenance/watchlist.md) — 追踪靶子（每条带当前值+日期+下一个触发点，按 🔴🟡🟢 优先级）
- [sweep-playbook.md](_maintenance/sweep-playbook.md) — 一次刷新扫描的标准动作（搜→比对→核真→落库→留痕→报 diff）
- [sweep-log.md](_maintenance/sweep-log.md) — 每次刷新的留痕
- 定时例行：`/schedule` 云端 agent 按周期跑 sweep-playbook（见下方日志）
